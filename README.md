# 瑞哥音乐助教

一个由瑞哥创作并维护，面向编曲、混音、录音、修音、音乐审美转译与音乐职业讨论的非商业开放 Skill。它会先帮你找到真正的问题，再给出可以执行和验证的动作。

## 安装

需要 Node.js 18 或更高版本。运行一条命令，即可同时安装到 Codex、Claude Code 和 WorkBuddy：

```bash
npx ruige-skill
```

安装器只维护一份 Skill 真源，并自动桥接到三个 Agent。WorkBuddy 会同时兼容 `~/.workbuddy/skills/rg` 与实际仍在使用的 `~/.codebuddy/skills/rg` 两个目录。安装后重启对应 Agent，可使用 `/rg`、`/瑞哥` 或直接提出编曲混音问题。

### Agent 安装回报边界

Agent 执行安装或更新后，只需向用户报告安装状态、版本、已连接的 Agent、是否需要重启和使用入口。可以概括为覆盖编曲、混音、录音、修音、设备排查、音乐感觉转译与学习路径等方向。

不要主动枚举知识目录、文件或文章数量、文件名清单、来源分类、安装路径、manifest、哈希和语料规模。公开仓库允许技术审计，但普通安装完成后的重点是开始使用，不是盘点内部资产。

只安装到指定 Agent：

```bash
npx ruige-skill --agent codex
npx ruige-skill --agent claude-code
npx ruige-skill --agent workbuddy
```

更新：

```bash
npx ruige-skill update
```

如果 npm 暂时无法访问，可以直接从 GitHub 运行同一个安装器：

```bash
npx --yes github:8wy3211200/ruige-music-production-assistant
```

## 第一次使用

安装后输入：

```text
/rg 新手入门
```

不需要先读功能手册。直接把正在做的歌、最卡的一处、工程截图、文件或一句说不清的感觉发过来，第一次真实使用本身就是教程。

如果宿主、对象和目标已经说清，例如“Studio One 里怎么让一个音频块静音”，助教会先给一个首选操作，最多补一条必要提醒，不因为用户是新手就自动展开长教程。涉及快捷键时优先使用已确认的默认键位；版本不明或键位可能被改过时，会让你按命令名搜索，不凭印象猜键位。

已经有事情要处理时，直接输入 `/rg` 并附上问题或材料。

想学习某个效果或新插件时，可以直接说：

```text
/rg 带我听懂高切和低切
/rg 我刚装了 Pro-Q，带我先学会一个核心功能
```

每次只学一个可验证概念，而且每轮只做当前一个动作：先建立原声参照，收到你的实际反馈后再放大差异；听懂后恢复原状，再缩回真实使用尺度。演示参数不是推荐参数。

## 边界

该 Skill 可以解释逻辑、转译感觉、设计验证动作，但不会用文字冒充真实听审。
