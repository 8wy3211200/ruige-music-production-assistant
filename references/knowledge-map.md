# 知识库检索地图

## 使用原则

- 先完成初步审题，再检索。不要让相似关键词把假问题带进旧答案。
- 每次优先读取最相关的 1-3 个文件，必须读正文和上下文。
- 案例参数只代表当时素材，不自动推广为固定标准。
- 可以用可靠通用知识补足未覆盖的概念、图示和 DAW 操作，但不得虚构瑞哥的经验或偏好。
- 不输出身份信息、联系方式、文件路径或无关原始对话。
- 不越过音频证据：可以说明如何验证，不得声称已经听出用户作品的问题。

## 来源与表达

- 完整使用可靠方法，默认只输出方法、条件和验证。
- 不展示参考文件名、目录、检索词或读取过程。
- 不把 AI 整理、推断或补充的内容归为瑞哥本人观点。
- 只有当前对话提供明确公开来源，或瑞哥在当前对话确认时，才说明具体归属。

## 知识库位置

知识库位于 Skill 内的 `knowledge/`。只检索当前 Skill 实际包含的文件。

先按问题类型读取最相关的 1-2 份汇总知识：

- `knowledge/公开基础知识.md`：问题校准、回答顺序和通用边界
- `knowledge/公开知识-编曲审美与执行.md`：审美拆解、配器职能、情绪推进和编曲说明
- `knowledge/公开知识-混音概念与信号流.md`：EQ、压缩、空间、插入、发送、总线和检查方法
- `knowledge/公开知识-录音修音与导出.md`：输入监听、Buffer、路由排查、修音和导出

其他补充知识按目录和文件的实际内容检索。

### 编曲审美与音乐语言转译

优先搜索参考方向、配器职能、结构、律动、动态和 AI 生成目标。没有覆盖时，用可靠通用知识继续解释。

### 混音概念与操作答疑

优先搜索概念、操作对象、信号流和真实目标，不只搜索“闷、糊、刺”等症状词。涉及具体工具或瑞哥是否使用过某工具时，没有可靠命中就说明这是通用建议。

## 搜索方法

优先组合对象与目标两个维度，例如：

```bash
rg -l -i --glob '*.md' '人声.*发送|发送.*人声' knowledge
rg -l -i --glob '*.md' '母带.*必须|必须.*母带' knowledge
rg -l -i --glob '*.md' 'Suno|AI.*编曲' knowledge
```

也可运行：

```bash
scripts/search-knowledge.sh '母带|总线压缩' 20
```

## 检索后判断

1. 文件回答的是同一个真问题，还是只有表面关键词相似？
2. 结论是否依赖听审、具体素材或特定软件？
3. 命中内容属于哪一级？
   - **稳定原理**：可以迁移，但仍说明适用条件。
   - **条件方法**：只在相同目标、对象和制作阶段下使用。
   - **案例参数**：只能作为实验起点，不能写成标准答案。
   - **个人经验或推荐**：只有明确公开来源或当前对话确认时才说明归属。
4. 当前回答是否越过听审边界？

检索命中不等于瑞哥本人推荐。没有可靠命中时，使用通用知识，不拼凑不相关案例。
