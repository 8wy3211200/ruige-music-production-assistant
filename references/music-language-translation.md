# 音乐感觉与专业语言双向转译

## 目标

让不懂术语的人听懂专业表达，也让只有直观感觉的人能把想法变成可执行的音乐选择。

不要把转译做成术语词典。真正要翻译的是声音、作用和选择之间的关系。

## 先判断用户需要哪一个方向

### 专业术语 → 直观感觉

用户看见或听见 `Pad`、`Pluck`、`Lead`、`Arp`、`Texture` 等词，却不知道它们在说什么时，按这个顺序解释：

1. **听起来怎样**：声音怎么出现、持续多久、明暗厚薄、宽窄远近、是否运动。
2. **在歌里负责什么**：主体、律动、和声、填充、低频、过渡还是气氛。
3. **怎么辨认**：给一个可以直接听或直接操作的判断方法。
4. **最后说术语**：说明这个词是在描述声音形态、演奏方式、音区、职能还是风格。

### 直观感觉 → 专业语言

用户说“想更空一点”“像有雾”“需要一点一闪一闪的东西”“副歌想突然打开”时，按这个顺序转译：

1. 保留用户原话，不急着贴风格名。
2. 找出他想改变的可感知差异：长短、疏密、明暗、软硬、远近、宽窄、动静、前后对比。
3. 判断需要哪种编曲职能参与。
4. 给出 1-3 个可能的音乐做法，并解释各自会带来什么感觉。
5. 必要时再补专业术语，方便用户搜索音色、操作 DAW 或与制作人沟通。

## 六个通用观察维度

遇到没入库的词，也先用这些维度拆解，不凭词面猜：

1. **起音**：一下就出来，还是慢慢浮现。
2. **持续**：短促消失，还是一直延续。
3. **音区与明暗**：偏低偏厚，还是偏高偏亮。
4. **空间**：集中或宽，靠前或靠后，干或有尾音。
5. **运动**：静止铺着、重复跳动、持续变化，还是只在节点出现。
6. **职能**：主体、律动、低频、和声、填充、过渡或气氛。

这些维度比预设名字可靠。同一个音色可以因为写法和处理不同而承担不同职能；不同乐器也可以承担相同职能。

## 角色优先于参数

先按听觉角色和编排作用解释 `Lead`、`BV`、`Dub`、八度等词，不把常见处理参数写进术语定义。

- `Lead` 是当前承担主要注意力和表达的声部。
- `BV` 是背景人声的上位职能，可以是和声、同度、呼应、填句或其他不承担主导叙事的声部，不固定等于不同音高的和声。
- `Dub` 和八度描述录法或音高关系时，仍需结合它在编排中承担主体、加厚还是背景的角色。

音量差、低切、高切或压缩量只能作为特定目标下的尝试，必须说明歌曲、段落、角色和听觉停止条件。通常 BV 的原则是“不抢主唱注意力、服务主体”；如果目标就是让听众明显听到合唱，BV 完全可以更突出，不能用固定小 `3-6 dB` 定义它。

## 用占位人声参与编曲

Beat 阶段的人声位置不只是一块等待 EQ 挖开的频段。优先放入自己录的临时哼唱、当前项目的参考干声或有权使用的占位素材，让它真实承担：

- 音区占位；
- 节奏和句尾空隙占位；
- 密度占位；
- 注意力占位。

围绕占位人声决定哪些乐器该减、该换音区、该错开节奏或该留白。等编曲关系已经成立、具体遮蔽确实出现后，再判断是否需要 EQ；不要在 Beat 尚未成型时猜未来要挖哪个频段。

## Pad 与 Pluck 示例

### Pad

先让用户感受：按下去后，声音通常不会立刻结束，而是持续铺在后面，常常比较宽、比较柔，不抢主旋律。

```text
Pad：  ━━━━━━━━━━━━━
       持续铺着，填住背景
```

它在编曲里常负责和声铺底、背景厚度和氛围。`Pad` 更接近对声音形态和职能的描述，不等于某一件固定乐器，也不等于“有混响的声音”。弦乐长音、处理过的钢琴或其他持续音色，也可能承担 Pad 的职能。

辨认方法：按住一个和弦，如果声音能持续托住背景，而且不会不断抢你的注意力，它就可能适合做 Pad。

### Pluck

先让用户感受：每个音一出来就有清楚的点，很快收掉；即使一直按着，也不会像 Pad 那样持续铺满。

```text
Pluck： •   •   •   •
        一个个短点，形成节奏和颗粒
```

它常负责点状节奏、分解和弦、旋律颗粒或高频细节。`Pluck` 更接近发声形态和演奏感觉，不代表一种固定风格。

辨认方法：连续弹几个八分音符。如果每一下边界清楚、音尾不长、能形成“点”的律动，它就接近 Pluck。

### 两者放在一起

```text
Pad：   ━━━━━━━━━━━━━  背景持续托住
Pluck： •  •  •  •  •   前面增加点状运动
```

不要只说“Pad 是铺底，Pluck 是拨弦”。要让用户知道：一个偏持续和背景，一个偏短促和点状；它们可以同时存在，也不是非此即彼。

## 术语不确定时

- 用户说的是听来的词，例如 `Plug`，不要直接假定就是 `Pluck`。
- 可以说：“你听到的可能是 Pluck，也可能是插件 Plug-in。它当时是在讲一种短促音色，还是在讲软件工具？”
- 一次只确认一个最影响解释的上下文：原句、截图、所在课程段落或它出现时的声音。

## 输出标准

好的转译至少让用户得到三样东西：

1. 下次听到或看到这个词，知道它大概在描述什么。
2. 回到音乐里，知道它承担什么作用，而不是只会背定义。
3. 回到 DAW，知道怎么辨认、搜索或试一次。

AI 提示词只在用户明确要生成音乐时提供。默认输出应服务于学习、编曲判断和与人沟通。
