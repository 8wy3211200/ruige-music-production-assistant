# 主人声和和声，应该先单轨精细处理，还是直接进 bus 一把处理？

**症状：** 用户在做主人声和和声时，容易纠结到底该先对每条轨道单独做 EQ / 压缩，还是先合并到 bus 上统一处理。

**原因：** 没有先按“它是不是一条完整的音轨线”来区分。主人声中那种一句一句错开录、逻辑上属于同一条线的内容，可以先打包；但和声如果本来就是多条独立层次，仍然应该先单独处理。

**解决：** 原则是先做单轨精细处理，再做总线整理。单个音轨线上的 EQ、压缩没问题；合并之后如果还想更整齐、控制动态，再在总线上补一层 EQ / 压缩。

**常见误区：** 还没分清层次就直接在 bus 上一把做完，结果细节不好控；或者反过来，明明属于同一条主人声线，也非要剪得特别碎。
