# VoiceMeeter直播内录耳返与双重监听排查

适用问题：
- OBS 只能收到麦克风，收不到伴奏或系统声音
- 我自己耳机里听着正常，直播间里却不是那回事
- 说话有双重监听、回声、拍子错位
- 想做系统内录、直播连麦、耳返，但越接越乱

核心结论：
- `A` 是给你自己听的，`B` 是给其他软件收的。很多直播问题，本质上都是这两个用途没分开。
- 双重监听通常不是“效果器有问题”，而是同一份声音被监听了两次。

最省事的直播/内录逻辑：
1. `A1` 只选你的耳机或监听音箱，负责你自己听。
2. 麦克风进 `IN1` 或 `IN2`。
3. Windows 默认播放设备设成 `Voicemeeter Input`，这样系统声音会进虚拟输入条。
4. 在 OBS、Discord、会议软件里，把输入源设成 `Voicemeeter Output` 或 `Voicemeeter Out B1`，这样软件收到的是 `B Bus` 的混合结果。
5. 自己监听尽量只保留一条路，不要同时开硬件直通、Windows 监听、应用内监听和 VoiceMeeter 监听。

为什么“自己听”和“观众听”经常不一样：
- 你耳机里听到的是 `A Bus` 监听视角。
- 直播、录屏、会议软件收到的是 `B Bus` 或虚拟输出视角。
- 这两路可以一样，也可以故意不一样。所以调试时不要只靠耳机判断，最好实际录一段或让 OBS 录一段回放。

最常见的几种故障：
- 只有麦克风，没有系统声。
  通常是直播软件输入还选着物理麦克风，而不是 `Voicemeeter Output / Out B1`。
- 只有系统声，没有麦克风。
  通常是麦克风那条没有送到 `B`，或者输入条根本没选对设备。
- 说话像有回声、自己听到两遍。
  常见原因是 Windows `Listen to this device` 开着，或声卡硬件直监听和 VoiceMeeter 软件监听同时在工作。
- 直播里有轻微拍子错位、小回声。
  如果你同时拿 `A1`、`A2` 去推不同物理输出，官方手册明确说它们不一定严格同步。普通用户最省事的做法是先只用一台监听设备。

排查双重监听最有效的顺序：
1. 先关 Windows `Listen to this device`。
2. 再关声卡自带的 Direct Monitor，确认不是硬件返听和软件返听叠加。
3. 再查 OBS、Discord 是否又把收到的声音返送回了你的耳机。
4. 最后再看 VoiceMeeter 里是不是把同一来源同时送到了你不需要的 Bus。

什么时候需要回放验证：
- 你自己耳机里舒服，但担心直播间太干、太湿、太吵。
- 你怀疑系统声和人声比例不对。
- 你怀疑路由对了，但观众端仍然奇怪。

- `VB-Audio Voicemeeter Standard User Manual` v1.1.2.2（DEC 2025）

一句话总结：
直播和内录别把所有监听都开着；先分清 `A` 是自己听、`B` 是给软件收，双重监听问题会少一大半。
