# 采样率位深和Buffer怎么选

适用问题：
- 采样率怎么选
- 位深怎么选
- Buffer Size 怎么调
- 为什么我录音会延迟、爆音、卡顿

核心结论：
- 大多数情况下，44.1kHz 或 48kHz 都够用。
- 位深优先 24bit，够稳也够常用。
- Buffer 不是越小越好，也不是越大越好，要看你是在录音还是在混音。

先说采样率和位深：
- 采样率可以理解成声音被记录得有多细。
- 位深可以理解成动态和精度层面的空间。
- 对大多数音乐制作、家录、修音、混音场景来说，44.1kHz/24bit 或 48kHz/24bit 已经足够。
- 没必要一上来追特别高的参数，反而会增加电脑负担。

什么时候用 44.1kHz：
- 常规音乐制作场景很多都够用。
- 如果你的项目主要就是做歌、发歌，这个设置很常见。

什么时候用 48kHz：
- 如果你还涉及视频、直播、一些跨平台使用场景，48kHz 也很常见。
- 关键不是哪个更高级，而是整个项目里尽量统一。

再说 Buffer Size：
- Buffer 小，延迟低，更适合录音、监听、实时演奏。
- Buffer 大，电脑更轻松，更适合后期混音、插件很多的工程。

如果你现在是在录音：
- 优先把 Buffer 调小一点。
- 因为你需要自己说话、唱歌、弹奏时听起来反应快。
- 但如果电脑开始爆音、卡顿，就要适当往上调。

如果你现在是在混音：
- 可以把 Buffer 调大一些。
- 因为这时候不太需要实时演奏，更需要系统稳定和插件带得动。

最常见的误区：
- 以为参数越高越专业。
- 以为 Buffer 永远调最小最好。
- 没有统一项目参数，后面导入导出、协作时容易出问题。

一句话总结：
采样率和位深够用、统一就行；Buffer 则按场景调，录音偏小，混音偏大。
