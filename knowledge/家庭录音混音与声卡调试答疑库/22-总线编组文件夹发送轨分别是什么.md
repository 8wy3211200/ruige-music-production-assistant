# 总线编组文件夹发送轨分别是什么

适用问题：
- 总线、编组、group、bus 是什么
- 文件夹和总线有什么区别
- 发送轨是干嘛的
- 为什么教程里老说信号流

核心结论：
- 总线、编组、group、bus，本质上都在说“把几条轨道统一到一个通道里处理”这件事。
- 文件夹主要是整理界面，不等于真正的处理通道。
- 发送轨是让一个声音把信号送到另一个效果通道里，常用于混响、延迟等共享效果。
- 先搞懂这些概念，比先背插件参数更重要。

怎么简单理解：

文件夹：
- 主要是为了整理轨道界面。
- 让你看起来不乱，方便收纳。
- 但文件夹本身通常不是一个真正的处理通道，不能直接代表统一的音频处理逻辑。

总线 / 编组 / bus / group：
- 就是把几条相关轨道合到一个通道里统一控制。
- 比如两条主唱、几条和声、一组鼓，都可以先编成一组。
- 这样你在这个总线上调音量、挂效果，就会同时影响这几条轨道。

发送轨：
- 是把原轨道的一部分信号“送”到另一个效果通道里。
- 最常见的用途就是混响和延迟。
- 这样做的好处是：不破坏原本的干声，同时多个轨道还能共用同一个效果。

为什么很多人会混：
- 因为不同宿主里名字不完全一样。
- 有的软件把 bus 和某些效果通道都叫 bus。
- 但名字不重要，重要的是你要知道：这个轨道到底是在整理、在统一处理，还是在接收发送。

为什么要学这个：
- 因为混音本质上不是一堆零散技巧，而是一套工作流。
- 如果你看不懂信号是怎么流动的，工程一复杂就很容易乱。
- 看懂信号流之后，很多工程里的问题都能自己定位。

一句话总结：
文件夹是整理界面，总线是统一处理，发送轨是把信号送去做共享效果。先把这三件事分清楚，很多宿主问题都会简单很多。
