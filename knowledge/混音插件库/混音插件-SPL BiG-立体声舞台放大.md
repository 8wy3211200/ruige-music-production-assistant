# SPL BiG 立体声舞台放大

## 核心概念
- `SPL BiG` 不是普通的 M/S 加宽器。官方手册明确说它处理的是由立体差信号衍生出来的内容，再经过滤波后混回原信号。
- 它最值得保留的价值是：即使把立体声做大，折叠回 mono 时仍然尽量保持干净，不靠明显相位问题换宽度。
- 它的操作也很刻意地被收窄成三件核心事：`Range` 选处理频段，`Stage` 决定这一段在舞台上前后/远近的感觉，`Bigness` 决定强度。
- 这意味着它不适合做特别精细的手术，但很适合快速回答“这个 pad / room / 总线能不能更开一点、更大一点”。

## 使用场景
- 优先用在已经有立体声信息的 synth、room、reverb、总线和母带微调。
- 适合那些“不是单纯想更宽，而是想让空间更大、更有舞台感”的场景。
- 适合总线末端少量定型，也适合单独的立体声音色轨。
- 不该先用在本来就接近单声道的素材上，那样可处理的信息很少。
- 也不该把它当救命工具去修相位已经很差、低频已经很散的混音。

## 核心参数
- 如果是小白，先动两个参数：`Bigness` 和 `Range`。
- `Bigness`：决定立体声舞台放大的强度。它越大，不只是更宽，也更容易把不稳定的侧边一起放大。
- `Range`：决定哪一段频率被重点处理。很多“越宽越乱”的问题，其实不是强度太大，而是频段选错了。
- `Stage`：控制被选中的那段频率在空间中的前后感和舞台位置。它比普通 widening 更像“推舞台”。
- `Bass`：额外低频增强，类似一个附加的被动低频加成。只在底盘已经稳的时候再考虑，不然容易误判成“更好”。
- `Mono Maker`：插件版的关键补强。低频一旦因拓宽开始松散，先用它把下方收回 mono。
- `Auto Gain`：对比前后时很有用，避免因为更响就误以为更宽更好。
- `Correlation`：这里比宽度数值更重要。如果相关性明显掉太多，说明不是“更大”，而是“更危险”。

## 实战案例
- 合成器 pad 太贴中间：先用中高频范围做少量 `Bigness`，再用 `Stage` 找前后感，通常比直接全频加宽更自然。
- 总线一拉宽就觉得底盘散：不要先退全部宽度，先开 `Mono Maker` 收低频，再决定要不要减 `Bigness`。
- Room / ambience 想更包裹：优先处理中高频区，不要把 kick 和 bass 所在区一起拉开。
- 一开就觉得爽，折成 mono 发现没精神：这时先看 `Correlation`，再检查是不是 `Range` 覆盖太低或 `Bass` 开过头。
